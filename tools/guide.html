Explorer's Field Guide

A cartographer's notes on the hidden mechanics of this site

The Living Geometry

Behind every page of this site lives a wireframe sphere. Not a static model, but a continuously deforming surface driven by 3D simplex noise. The mesh is built from a grid of 60 latitude lines by 60 longitude lines -- 3,600 vertices in total, recalculated every frame.

Each vertex sits on a unit sphere, then gets displaced outward by a noise function that samples three-dimensional space. The noise field itself drifts over time, so the surface never repeats. The result is something that looks organic -- like a deep-sea creature caught mid-thought.

The wireframe renders using quadratic Bezier curves rather than straight line segments. This gives the mesh a smoother appearance even at lower resolutions, and makes the whole shape feel less like a polygon and more like a living membrane.

The Scroll Current

The blob responds to scrolling. Scroll fast enough and the geometry spins with you -- the Y-axis rotation tracks scroll velocity, so quick flicks send it whirling. Stop, and it breathes deeper. The idle morph speed is set higher than the active speed, which means the shape actually moves more when you are still.

Each section of the main page assigns a different hue to the wireframe. The transitions between them are not instant -- the color lerps smoothly from one value to the next using linear interpolation on the RGB channels. Scroll from the hero into the first featured project and watch the purple bleed into deep pink. The section detection fires a custom sectionChanged event on the window, and the animation system begins interpolating toward the new target. If you have set a hue override through the control panel, the section colors are bypassed entirely.

Prismatic Override

There is something hidden at the far end of the hue slider.

[click to reveal]

Drag the hue slider to its maximum value -- 360 degrees -- and hold it there. After roughly 1.6 seconds, the wireframe ignites.

Rainbow mode. Each latitude and longitude line of the wireframe gets its own hue, cycling rapidly through the spectrum. A pulsing glow radiates outward. The control panel caret itself glows with a shifting rainbow border.

This state is session-only. Refresh the page and it vanishes. It was never meant to be permanent -- just a reward for the curious.

To deactivate, drag the hue slider away from the maximum, or hit "Reset to Defaults." The blob returns to its section-controlled colors as if nothing happened.

Traffic Therapy

Traffic Therapy is a sliding block puzzle in the tradition of Rush Hour. A grid of interlocking blocks, one of which -- the target -- needs to reach the exit. Blocks slide along their axis only: horizontal blocks move left and right, vertical blocks move up and down. The 1x1 target block moves in any direction.

Five hand-crafted levels, each verified solvable by a built-in BFS solver. The solver explores every possible board state to find the optimal solution, which sets the par for each level. Your move count determines your star rating -- four stars if you beat par, one star if you take the scenic route.

Progress persists via cookies. Star ratings only go up, never down. Replaying a level with a worse score does not overwrite your best.

Admin Mode

On the title screen, there is a subtle element near the title text. It does not look clickable. Click it anyway.

A prompt appears: are you god?

Type ya.

An "ADMIN" badge appears in the corner. All levels unlock immediately. A new "Level Editor" button materializes on the level select screen.

The editor is a full authoring tool. Set the grid size, place blocks from a toolbar, drag them into position, designate the exit. A "Test" button runs the BFS solver against your creation and reports whether it is solvable and the optimal move count. You can play-test your level without leaving the editor. Export individual levels or the entire set as JSON.

Admin mode does not persist. Refresh, and it is gone. The puzzle resets to civilian access.

Dodgers Win Deal Notifier

Dodgers win at home? $6 meal deal at Panda Express. Dodgers throw 7 or more strikeouts? Free Jumbo Jack at Jack in the Box (with purchase of a large drink). This is the most important public service on this entire website.

Sign up with your phone number and get a text whenever your local fast food joints turn on the free food faucet. No app to install, no account to create -- just a Google Form and an automated pipeline that watches the box scores so you do not have to.

Public Art Submission

The Public Art Submission Form is a mobile-friendly tool for documenting street art, murals, and other interesting things you find while wandering around. It grabs your photo, tags your location, and sends the data to a shared collection.

This is part of a larger project in progress -- a full public art map that aggregates submissions and lets you browse street art by neighborhood. The submission form is the data collection layer. A standalone app, Street Art Hunter, is also in development as a dedicated mobile experience for contributing to the map.

If you spot something worth documenting, submit it. Every entry helps build the dataset.

eBay Lookup Chrome Extension

A Chrome extension that adds eBay price lookups directly into Craigslist listings. See what the same item is selling for on eBay without leaving the page -- useful for figuring out whether that "great deal" is actually a great deal.

The extension requires an eBay API key to function. Head to the GitHub repository for installation instructions. If you actually want to give it a shot, contact me for an API key.

620,000 Collisions

The LA Collisions Dashboard visualizes twelve years of LAPD collision records -- every reported incident from 2010 through 2021. The raw dataset is 303 megabytes of JSON.

Serving that much data to a browser without a backend required some engineering. The dataset is preprocessed into 421 spatial tiles, each covering a geographic region. As you pan and zoom the map, only the relevant tiles load. The spatial index uses RBush, a high-performance R-tree implementation, to determine which tiles intersect the current viewport.

The map renders through MapLibre GL, an open-source vector tile renderer. Temporal analysis charts use Chart.js. There is no server processing, no database queries, no API calls. Everything runs in the browser, against static JSON files.

The data is the data. The dashboard just makes it possible to see.
